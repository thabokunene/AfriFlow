# coverage/

This directory holds per-module coverage breakdown reports generated during CI
pipeline runs and local `make coverage` invocations.

---

## Contents

| File | Format | Generated by | Purpose |
|------|--------|--------------|---------|
| `forex_ingestion_coverage.md` | Markdown table | `scripts/cell_coverage_report.py` | Line / branch / function coverage for `domains/forex/ingestion/` |
| `forex_ingestion_coverage_summary.json` | JSON | `scripts/cell_coverage_report.py` | Machine-readable version of the above; consumed by downstream tooling |
| `flow_drift_detector_coverage.md` | Markdown table | `scripts/cell_coverage_report.py` | Line / branch / function coverage for `cib/processing/flink/flow_drift_detector.py` |
| `flow_drift_detector_coverage_summary.json` | JSON | `scripts/cell_coverage_report.py` | Machine-readable version of the above |
| `coverage_audit.md` | Markdown | Manual / audit tooling | Full audit log of this directory — issues found, fixes applied, residual risks |
| `README.md` | Markdown | Manual | This file |

> The following files are generated only during pull-request builds and will not
> be present in a local checkout unless you ran `make coverage` with cell ingestion
> changes staged:
>
> - `cell_ingestion_coverage.md`
> - `cell_ingestion_coverage_summary.json`
> - `changed_files.json`
> - `gating_log.json`
> - `benchmark_report.json`

## How to regenerate

Run from the `afriflow/` directory **after** executing the test suite so that
`coverage.xml` and `coverage.json` are up to date:

```bash
# Full test run + coverage files
python -m pytest -q --cov=domains --cov-branch \
    --cov-report=xml --cov-report=json

# Forex ingestion breakdown
python scripts/cell_coverage_report.py \
    --xml coverage.xml --json coverage.json \
    --include-subpath forex/ingestion \
    --line-min 90 --branch-min 0 --func-min 0 \
    --write-md  coverage/forex_ingestion_coverage.md \
    --write-json coverage/forex_ingestion_coverage_summary.json

# CIB FlowDriftDetector breakdown
python scripts/cell_coverage_report.py \
    --xml coverage.xml --json coverage.json \
    --include-subpath cib/processing/flink \
    --include-file-pattern "flow_drift_detector\.py" \
    --line-min 90 --branch-min 0 --func-min 0 \
    --write-md  coverage/flow_drift_detector_coverage.md \
    --write-json coverage/flow_drift_detector_coverage_summary.json
```

## Path resolution note

`coverage.xml` is generated with **two** source roots:

```
<source>…/afriflow</source>
<source>…/afriflow/domains</source>
```

Because `pytest-cov` resolves `<class filename>` relative to the *second* source,
paths in the XML are **domain-relative** (e.g. `forex/ingestion/kafka_producer.py`),
**not** repo-relative (e.g. `domains/forex/ingestion/kafka_producer.py`).

All `--include-subpath` arguments and the `coverage_subpath` keys in
`config/coverage_gates.json` must therefore omit the `domains/` prefix.
The `path` field in `coverage_gates.json` retains the `domains/` prefix because it
is used by `gating_framework.py` to match git-diff paths, which are repo-relative.

## Thresholds

| Module | Line min | Branch min | Function min |
|--------|----------|------------|--------------|
| `cell/ingestion` | 90% | 90% | 90% |
| `forex/ingestion` | 90% | — | — |
| `cib/processing/flink` (flow_drift_detector only) | 90% | — | — |

Gates are defined in `config/coverage_gates.json` and enforced by
`scripts/gating_framework.py` (PR builds) and directly in `ci-tests.yml`
(every build).
