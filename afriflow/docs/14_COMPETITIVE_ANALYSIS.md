This document compares AfriFlow to competitive
approaches such as traditional data warehouses,
standalone analytics platforms, and single-domain
solutions.

## Competitor Categories

- **Traditional DW/BI**: Centralised data warehouse
  with batch ETL.
- **Domain-specific platforms**: Solutions focussed on
  one line of business (e.g., treasury, insurance).
- **Point solutions**: Analytics or alerting tools that
  operate on a single data source.

## AfriFlow Differentiators

- Real time cross domain correlation.
- Federated data residency.
- Golden ID entity graph.
- Signal publishing to operational systems.

## Risks

- Data governance complexity.
- Integration effort across legacy systems.
- Need for organisational alignment and change
  management.
